Objectif du Système d'Affectation

Le système TutorMatch implémente trois algorithmes distincts pour résoudre le problème d'affectation optimale de tuteurs aux étudiants en stage. Chaque algorithme répond à des besoins spécifiques selon la taille du problème, les contraintes et les objectifs d'optimisation.

Problème d'Affectation

Assigner N étudiants à M tuteurs en optimisant satisfaction et contraintes

Contraintes Multiples

Capacité des tuteurs, préférences, spécialisations et équilibrage

Optimisation

Maximiser la satisfaction globale et minimiser les conflits

Algorithme Glouton (Greedy)

Stratégie de sélection des meilleures options locales

Complexité: O(n² log n) Rapide Efficace en mémoire

Principe de Fonctionnement

L'algorithme glouton suit une stratégie simple : à chaque étape, il fait le choix localement optimal sans reconsidérer les décisions précédentes. Pour l'affectation de tuteurs, cela signifie :

  1. Calcul des scores : Évaluation de la compatibilité entre chaque étudiant et chaque tuteur
  2. Tri par priorité : Classement des associations par score décroissant
  3. Affectation séquentielle : Attribution en suivant l'ordre de priorité, en vérifiant les contraintes
  4. Mise à jour : Actualisation des capacités disponibles après chaque affectation

Formule de Calcul du Score

Score(étudiant, tuteur) =
α × CompatibilitéDomaine + β × PreférenceTuteur + γ × ChargeEquilibrage
Où α = 0.5, β = 0.3, γ = 0.2 (poids configurables)

Pseudocode 

ALGORITHME GreedyAssignment(étudiants, tuteurs)
DÉBUT
    // Initialisation
    affectations ← {}
    scores ← []
    
    // Calcul de tous les scores
    POUR chaque étudiant e DANS étudiants FAIRE
        POUR chaque tuteur t DANS tuteurs FAIRE
            score ← calculerScore(e, t)
            scores.ajouter({étudiant: e, tuteur: t, score: score})
        FIN POUR
    FIN POUR
    
    // Tri par score décroissant
    scores ← trier(scores, DÉCROISSANT)
    
    // Affectation gloutonne
    POUR chaque candidat DANS scores FAIRE
        SI tuteur.capacitéDisponible > 0 ET étudiant.nonAffecté ALORS
            affecter(candidat.étudiant, candidat.tuteur)
            tuteur.capacitéDisponible -= 1
            étudiant.nonAffecté ← FAUX
        FIN SI
    FIN POUR
    
    RETOURNER affectations
FIN

Algorithme Hongrois (Hungarian)

Solution optimale pour les problèmes d'affectation bipartite

Complexité: O(n³) Optimal Théoriquement prouvé

Principe Mathématique

L'algorithme hongrois, développé par Harold Kuhn en 1955, résout optimalement le problème d'affectation en transformant le problème en un problème de couplage parfait de poids minimum dans un graphe bipartite.

Objectif : Minimiser
∑(i,j) c[i][j] × x[i][j]
Où c[i][j] est le coût d'affecter l'étudiant i au tuteur j
et x[i][j] ∈ {0,1} indique l'affectation

Le principe repose sur le théorème de Koenig-Egerváry qui établit une dualité entre couplage maximum et couverture minimum dans les graphes bipartites.

Étapes de l'Algorithme

Phase 1 : Préparation
  1. Construction de la matrice de coût
  2. Transformation max → min (si nécessaire)
  3. Équilibrage de la matrice (n×n)
Phase 2 : Réduction
  1. Soustraction du minimum de chaque ligne
  2. Soustraction du minimum de chaque colonne
  3. Obtention de zéros dans la matrice
Phase 3 : Couverture
  1. Marquage des zéros indépendants
  2. Couverture par lignes et colonnes
  3. Test d'optimalité
Phase 4 : Optimisation
  1. Création de nouveaux zéros
  2. Amélioration du couplage
  3. Itération jusqu'à l'optimum

Pseudocode Détaillé 

ALGORITHME HungarianAssignment(matriceCost)
DÉBUT
    n ← taille(matriceCost)
    
    // Phase 1: Réduction par lignes
    POUR i ← 0 À n-1 FAIRE
        minLigne ← min(matriceCost[i])
        POUR j ← 0 À n-1 FAIRE
            matriceCost[i][j] -= minLigne
        FIN POUR
    FIN POUR
    
    // Phase 2: Réduction par colonnes
    POUR j ← 0 À n-1 FAIRE
        minColonne ← min(colonne[j])
        POUR i ← 0 À n-1 FAIRE
            matriceCost[i][j] -= minColonne
        FIN POUR
    FIN POUR
    
    RÉPÉTER
        // Phase 3: Couplage maximum
        couplage ← trouveCouplagMaximum(matriceCost)
        
        SI taille(couplage) = n ALORS
            RETOURNER couplage  // Solution optimale trouvée
        FIN SI
        
        // Phase 4: Amélioration
        couverture ← trouveCouvertureMinimum(matriceCost)
        theta ← min(éléments non couverts)
        
        // Mise à jour de la matrice
        POUR i ← 0 À n-1 FAIRE
            POUR j ← 0 À n-1 FAIRE
                SI ligne[i] couverte ET colonne[j] non couverte ALORS
                    matriceCost[i][j] += theta
                SINON SI ligne[i] non couverte ET colonne[j] couverte ALORS
                    matriceCost[i][j] -= theta
                FIN SI
            FIN POUR
        FIN POUR
    JUSQU'À solution trouvée
FIN

Théorème de Koenig-Egerváry

Dans un graphe bipartite, la taille maximum d'un couplage est égale à la taille minimum d'une couverture par sommets.

Preuve de Correctitude
  1. Invariant : À chaque itération, nous maintenons une solution réalisable du dual
  2. Terminaison : L'algorithme termine en temps polynomial
  3. Optimalité : Quand l'algorithme termine, les conditions d'optimalité de Kuhn-Munkres sont satisfaites

Algorithme Génétique (Genetic)

Optimisation évolutionnaire inspirée de la sélection naturelle

Complexité: O(g×p×n) Adaptatif Multi-contraintes

Principe Évolutionnaire

L'algorithme génétique simule l'évolution naturelle pour résoudre des problèmes d'optimisation complexes. Il maintient une population de solutions candidates qui évoluent au fil des générations par sélection, croisement et mutation.

Population

Ensemble de solutions candidates

Fitness

Évaluation de la qualité

Croisement

Combinaison de solutions

Mutation

Diversification génétique

Représentation des Chromosomes

Chaque solution (chromosome) est représentée comme un vecteur d'affectations :

Chromosome = [tuteur_étudiant_1, tuteur_étudiant_2, ..., tuteur_étudiant_n]

Exemple pour 5 étudiants et 3 tuteurs :
Chromosome = [2, 1, 3, 1, 2]
- Étudiant 1 → Tuteur 2
- Étudiant 2 → Tuteur 1  
- Étudiant 3 → Tuteur 3
- Étudiant 4 → Tuteur 1
- Étudiant 5 → Tuteur 2
Fonction de Fitness Complexe
F(chromosome) = w₁×S + w₂×C + w₃×E + w₄×P - w₅×V
S = Satisfaction, C = Contraintes, E = Équilibrage
P = Préférences, V = Violations

Opérateurs Génétiques

Sélection

Sélection par Tournoi :

  • Sélection de k individus aléatoirement
  • Choix du meilleur fitness
  • Pression de sélection ajustable
POUR i ← 1 À k FAIRE
    candidat ← aléatoire(population)
    SI fitness(candidat) > fitness(meilleur) ALORS
        meilleur ← candidat
    FIN SI
FIN POUR
Croisement

Croisement Uniforme :

  • Masque binaire aléatoire
  • Échange basé sur le masque
  • Préservation de la diversité
masque ← générerMasqueBinaire(n)
POUR i ← 0 À n-1 FAIRE
    SI masque[i] = 1 ALORS
        enfant1[i] ← parent2[i]
        enfant2[i] ← parent1[i]
    FIN SI
FIN POUR
Mutation

Mutation Adaptative :

  • Taux variable selon convergence
  • Respect des contraintes
  • Réparation automatique
POUR chaque gène FAIRE
    SI aléatoire() < tauxMutation ALORS
        nouveauTuteur ← aléatoire(tuteursDisponibles)
        gène ← nouveauTuteur
        réparer(chromosome)
    FIN SI
FIN POUR

Algorithme Principal 

ALGORITHME GeneticAssignment(étudiants, tuteurs, paramètres)
DÉBUT
    // Initialisation
    population ← générerPopulationInitiale(paramètres.taille)
    génération ← 0
    meilleuresSolutions ← []
    
    RÉPÉTER
        // Évaluation
        fitness ← []
        POUR chaque individu DANS population FAIRE
            fitness.ajouter(évaluerFitness(individu))
        FIN POUR
        
        // Sélection de la meilleure solution
        meilleur ← individu avec fitness maximum
        meilleuresSolutions.ajouter(meilleur)
        
        // Critère d'arrêt
        SI (génération ≥ paramètres.maxGénérations) OU 
           (convergence(meilleuresSolutions)) ALORS
            SORTIR DE LA BOUCLE
        FIN SI
        
        // Nouvelle génération
        nouvellePopulation ← []
        
        // Élitisme : garder les meilleurs
        nombreÉlites ← paramètres.taille × paramètres.tauxÉlitisme
        élites ← sélectionnerMeilleurs(population, fitness, nombreÉlites)
        nouvellePopulation.ajouter(élites)
        
        // Croisement et mutation
        TANT QUE taille(nouvellePopulation) < paramètres.taille FAIRE
            parent1 ← sélectionTournoi(population, fitness)
            parent2 ← sélectionTournoi(population, fitness)
            
            SI aléatoire() < paramètres.tauxCroisement ALORS
                (enfant1, enfant2) ← croisement(parent1, parent2)
            SINON
                enfant1, enfant2 ← parent1, parent2
            FIN SI
            
            SI aléatoire() < paramètres.tauxMutation ALORS
                enfant1 ← mutation(enfant1)
                enfant2 ← mutation(enfant2)
            FIN SI
            
            nouvellePopulation.ajouter(enfant1, enfant2)
        FIN TANT QUE
        
        population ← nouvellePopulation
        génération += 1
    JUSQU'À critère d'arrêt
    
    RETOURNER meilleur
FIN

Critères de Convergence et Arrêt
Critères d'Arrêt Multiples
  1. Nombre maximum de générations : Limite stricte pour éviter les calculs excessifs
  2. Stagnation du fitness : Arrêt si aucune amélioration sur N générations
  3. Convergence de population : Arrêt si la diversité devient trop faible
  4. Fitness cible atteint : Arrêt si un seuil de qualité est atteint
Métriques de Convergence
  • Diversité génotypique : Mesure de la variété dans la population
  • Amélioration relative : Pourcentage d'amélioration par génération
  • Stabilité du fitness : Variance du fitness dans la population

Implémentation Glouton Optimisée 

validateInputs($students, $tutors);
        $startTime = microtime(true);
        
        // Calcul de tous les scores de compatibilité
        $compatibilities = [];
        foreach ($students as $student) {
            foreach ($tutors as $tutor) {
                if ($this->isValidAssignment($student, $tutor, $constraints)) {
                    $score = $this->calculateCompatibilityScore($student, $tutor);
                    $compatibilities[] = new Compatibility($student, $tutor, $score);
                }
            }
        }
        
        // Tri par score décroissant (utilisation d'un heap pour optimiser)
        $heap = new SplMaxHeap();
        foreach ($compatibilities as $compatibility) {
            $heap->insert($compatibility);
        }
        
        // Affectation gloutonne
        $assignments = [];
        $assignedStudents = [];
        $tutorLoads = array_fill_keys(array_column($tutors, 'id'), 0);
        $tutorCapacities = array_column($tutors, 'maxStudents', 'id');
        
        while (!$heap->isEmpty()) {
            $compatibility = $heap->extract();
            $student = $compatibility->getStudent();
            $tutor = $compatibility->getTutor();
            
            // Vérifier que l'étudiant n'est pas déjà affecté
            if (in_array($student->getId(), $assignedStudents)) {
                continue;
            }
            
            // Vérifier la capacité du tuteur
            if ($tutorLoads[$tutor->getId()] >= $tutorCapacities[$tutor->getId()]) {
                continue;
            }
            
            // Effectuer l'affectation
            $assignments[] = new Assignment($student, $tutor, $compatibility->getScore());
            $assignedStudents[] = $student->getId();
            $tutorLoads[$tutor->getId()]++;
            
            $this->logger->debug('Student {student} assigned to tutor {tutor}', [
                'student' => $student->getId(),
                'tutor' => $tutor->getId(),
                'score' => $compatibility->getScore()
            ]);
        }
        
        $executionTime = microtime(true) - $startTime;
        
        return new AssignmentResult($assignments, [
            'algorithm' => 'Greedy',
            'executionTime' => $executionTime,
            'assignedStudents' => count($assignments),
            'totalStudents' => count($students),
            'averageScore' => $this->calculateAverageScore($assignments)
        ]);
    }

    public function performAssignment(int $sessionId, string $algorithmType = 'auto'): AssignmentResult
    {
        // Détermine l'algorithme optimal
        if ($algorithmType === 'auto') {
            $algorithmType = $this->selectOptimalAlgorithm($sessionId);
        }
        
        if (!isset($this->algorithms[$algorithmType])) {
            throw new InvalidArgumentException("Unknown algorithm: {$algorithmType}");
        }
        
        // Cache key basé sur la session et l'algorithme
        $cacheKey = "assignment:{$sessionId}:{$algorithmType}";
        
        // Vérifier le cache
        if ($this->cache->has($cacheKey)) {
            return $this->cache->get($cacheKey);
        }
        
        // Charger les données
        $students = $this->loadStudents($sessionId);
        $tutors = $this->loadTutors($sessionId);
        $constraints = $this->loadConstraints($sessionId);
        
        // Exécuter l'algorithme
        $algorithm = $this->algorithms[$algorithmType];
        $result = $algorithm->assign($students, $tutors, $constraints);
        
        // Sauvegarder en base
        $this->saveAssignmentResult($sessionId, $result);
        
        // Mettre en cache (TTL: 1 heure)
        $this->cache->set($cacheKey, $result, 3600);
        
        return $result;
    }
    
    private function selectOptimalAlgorithm(int $sessionId): string
    {
        $studentCount = $this->db->query(
            'SELECT COUNT(*) FROM students WHERE session_id = ?', 
            [$sessionId]
        )->fetchColumn();
        
        $complexityLevel = $this->analyzeConstraintComplexity($sessionId);
        
        // Logique de sélection automatique
        if ($studentCount < 50) {
            return $complexityLevel > 0.7 ? 'genetic' : 'hungarian';
        } elseif ($studentCount < 200) {
            return $complexityLevel > 0.5 ? 'genetic' : 'greedy';
        } else {
            return 'genetic'; // Meilleur passage à l'échelle
        }
    }
    
    private function analyzeConstraintComplexity(int $sessionId): float
    {
        // Analyse de la complexité des contraintes
        $metrics = $this->db->query(
            'SELECT 
                AVG(max_students) as avg_capacity,
                COUNT(DISTINCT domain) as domain_diversity,
                COUNT(*) as tutor_count
            FROM tutors WHERE session_id = ?',
            [$sessionId]
        )->fetch();
        
        // Score de complexité normalisé (0-1)
        $capacityVariation = $this->calculateCapacityVariation($sessionId);
        $domainConcentration = $metrics['domain_diversity'] / $metrics['tutor_count'];
        
        return ($capacityVariation * 0.6) + ($domainConcentration * 0.4);
    }
}

            $startTime = microtime(true);
            $result = $this->algorithm->assign($students, $tutors);
            $executionTime = microtime(true) - $startTime;
            
            // Assertions de performance
            $this->assertLessThan(
                $this->getExpectedMaxTime($size),
                $executionTime,
                "Algorithm too slow for size {$size}"
            );
            
            $this->assertGreaterThan(
                0.8, // Au moins 80% de satisfaction
                $result->getAverageScore(),
                "Quality too low for size {$size}"
            );
        }
    }
    
    private function getExpectedMaxTime(int $size): float
    {
        // Temps maximum attendu basé sur la complexité O(n² log n)
        return 0.001 * $size * $size * log($size);
    }
}

        ]);

    public function generatePerformanceReport(int $sessionId): array
    {
        return [
            'session_id' => $sessionId,
            'algorithms_used' => $this->getAlgorithmsUsed($sessionId),
            'average_execution_time' => $this->getAverageExecutionTime($sessionId),
            'quality_metrics' => $this->getQualityMetrics($sessionId),
            'constraint_satisfaction' => $this->getConstraintSatisfaction($sessionId),
            'recommendations' => $this->generateRecommendations($sessionId)
        ];
    }
    
    private function generateRecommendations(int $sessionId): array
    {
        $metrics = $this->getQualityMetrics($sessionId);
        $recommendations = [];
        
        if ($metrics['satisfaction_rate'] < 0.85) {
            $recommendations[] = 'Consider using Hungarian algorithm for better quality';
        }
        
        if ($metrics['execution_time'] > 5.0) {
            $recommendations[] = 'Consider using Greedy algorithm for faster execution';
        }
        
        if ($metrics['constraint_violations'] > 0.05) {
            $recommendations[] = 'Review constraint configuration or use Genetic algorithm';
        }
        
        return $recommendations;
    }
